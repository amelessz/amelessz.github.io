数据库管理模块

业务需求

  • 在实际工作中，最重要的时当前数据，n天前的历史数据价值下降。

  • 如果表的记录超过亿，插入和查询的性能略有下降，迁移和备份也不方便。

  • 如果历史数据不需要保存，可以删除它；如果历史数据仍有价值，可以把若干天之前的历史数据备份到历史表中。

    • 如果数据不是特别大，可以只用一个历史表如T_ZHOBTMIND T_ZHOBTMIND_HIS
    • 如果数据量特别大，可以一年一个历史表，或一个月一个历史表，用年份或月份当后缀。
    • 删除语句deletr from T_ZHOBTMIND where ddatetime<sysdate-7;
      • 用这种方式太简单粗暴，如果数据量特别大，会产生大事物，冲击数据库系统。
      • 正确做法是采用蚂蚁搬家似的方法，每执行一次SQL语句只操作数百行记录。

数据库管理有两个子模块

  • 数据清理：从表中删除指定条件的记录。
  • 数据迁移：把一个表中指定条件的记录搬到另一个结构相同的表（从当前表迁移到历史表）

数据管理-数据清理

主要逻辑：

连接数据库后准备一个临时变量存放每次从结果集读取到的值

char tempvalue[21]; // 暂存本次从结果集读取到的值

准备查询语句和查询的操作对象

    // 准备查询满足条件的sql语句
    // select rowid from tablename where ();
    sqlstatement stmtsel(&conn); // 查询的数据库操作对象
    string selsql = sformat("select %s from %s %s", starg.keycol, starg.tname, starg.where);
    stmtsel.prepare(selsql);
    stmtsel.bindout(1, tempvalue, 20);	//绑定输出到存放结构的临时变量中

准备删除语句

// delete  from tablename where rowid in (:1,:2,:3,:4...max)
    string delsql = sformat("delete from %s where %s in (", starg.tname, starg.keycol);

    for (int i = 0; i < starg.maxcount; i++)
        delsql = delsql + sformat(":%lu,", i + 1);
    deleterchr(delsql, ',');
    delsql = delsql + ")";

绑定要删除的值，最大值starg.maxcount，表示每次删除的最大数量

for (int i = 0; i < starg.maxcount; i++)
