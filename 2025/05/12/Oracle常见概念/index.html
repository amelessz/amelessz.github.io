    数据同步 - 基本概念

    单实例 Oracle (Single-Instance Oracle)

    • 定义与部署:
      • 安装在本地虚拟机、云服务器、物理服务器上的 Oracle 都属于单实例。
    • 单点故障 (Single Point of Failure - SPOF):
      • 概念: 单点故障（英语：single point of failure，缩写 SPOF）是指系统中一点失效，就会让整个系统无法运作的部件，换句话说，单点故障即会整体故障。
      • 问题: 单实例的 Oracle 存在单点故障。

    Oracle RAC (Real Application Clusters)

    • 概念与目标:
      • 译名: 实时应用集群。
      • 核心描述: Oracle RAC 是 Oracle 数据库的一个选件（Option），可以部署在多个服务器节点上，共享同一个数据库（通常位于共享存储如 ASM、SAN 或 NAS 上）。这些节点对外表现为一个统一的数据库服务，但在物理上是多个 Oracle 实例在协同工作。
      • 主要目的: 某个服务器出现故障时，其他节点仍正常工作，服务自动切换，不会对业务造成影响。
    • 核心优势：高可用性 (High Availability):
      • 解释: 只要一个实例存活，就可以提供服务，确保业务连续性。
      • 性能考量: 数据库的性能瓶颈通常在磁盘 I/O，因此 RAC 一般推荐两个节点以获得最佳的高可用性与性能平衡。
    • RAC 的软硬件要求:
      • 服务器:
        • 至少需要两台高性能服务器，或者 IBM 小型机的分区（操作系统通常是 AIX）。
        • 高性能服务器代表：IBM 3850（高端），IBM 3650（中端）。
        • IBM 小型机可以分为多个区，每个区都可以视为一台高性能服务器。
      • 存储设备:
        • 需要高性能存储设备，通常自带容错机制，不存在单点故障。
      • 软件支持:
        • 需要购买 Oracle RAC 软件许可（通常价格是单实例的两倍）。
    • RAC的性能
      • RAC是高可用方案，不是高性能方案。
      • 数据库的性能瓶颈在磁盘I/O，多个服务器不能提升性能。
      • RAC的各实例之间协调需要消耗资源，所以性能比单实例略低。

    Oracle的Data Guard

    • 概念：

      把主数据库的日志传输到备用数据库，在备用数据库重做主数据库的任何操作，所有其内容和主数据库一样。主数据库出现故障可以自动切换为备用数据库。

    • Data Guard的特点：

      • Data Guard主要用于(异地)数据容灾和读写分离。
      • 备用数据库是主数据库的副本，在主数据库出现故障时，可代替主数
        据库。

    • Data Guard的应用

      • RAC已足够安全，普通的、非钱命关天的项目，一般不使用Data
        Guard.
      • 备用数据库的数据量和主数据库一样大，也就是说，要求的存储空间
        一样大，成本很高。
      • 主数据库故障时，备用数据库可自动切换成主数据库，但是，故障难
        以鉴定。

    Oracle的GoldenGate（OGG）

    • 概念：

      Oracle GoldenGate (OGG) 是一种用于实时、异构数据复制和集成的软件。它通过读取源数据库的事务日志来捕获数据变化，并将这些变化以极低的延迟投递到各种目标系统（可以是不同类型的数据库），从而实现数据同步、高可用、灾备、零停机迁移等功能。

    • OGG的特点：

      • 无法实现同步复制，延迟时间为若干秒。
      • 支持一对一、一对多、多对一、多对多和双向复制等。
      • 提供数据压缩和加密，降低传输所需的带宽，提高了安全性。
