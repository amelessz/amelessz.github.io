阻塞和非阻塞IO

发布于 2025-05-15 IO 

阻塞&非阻塞IO

  • 阻塞：在进/线程中，发起一个调用时，在调用返回之前，进/线程会
    被阻塞等待，等待中的进/线让出CPU的使用权。
  • 非阻塞：在进/线程中，发起一个调用时，会立即返回。
  • 会阻塞的四个函数：connect()、accept()、send()、recv()。
  • 在传统的网络服务端程序中（每连接每线/进程），采用阻塞IO。
  • 在IO复用的模型中，事件循环不能被阻塞在任何环节，所以，应该采用非阻塞IO。

非阻塞IO的函数

  • 对非阻塞的IO调用connect()函数,不论是否能连接成功，connect()都会立即返回失败，errno==EINPROGRESS

    将socket设置成非阻塞

    int setnonblocking(int fd)
&#123;
    int  flags;

    // 获取fd的状态。
    if  ((flags=fcntl(fd,F_GETFL,0))==-1)
        flags = 0;

    return fcntl(fd,F_SETFL,flags|O_NONBLOCK);
&#125;

  • 对非阻塞的IO调用connect()函数后，如果socket的状态是可写的，证明连接是成功的，否则是失败的。

    使用poll或epoll监听客户端socket的读事件，如果可写表示连接成功。

        pollfd fds;
    fds.fd=sockfd;
    fds.events=POLLOUT;
    poll(&fds,1,-1);
    if (fds.revents==POLLOUT)
        printf("connect ok.\n");
    else
        printf("connect failed.\n");

  • 对非阻塞的IO调用accept()，如果已连接队列中没有socket，函数立即
    返回失败，errno==EAGAIN

  • 对非阻塞的IO调用send()，如果socket不可写（发送缓冲区已满），函
    数立即返回失败，errno==EAGAIN
