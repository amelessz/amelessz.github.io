asio

Boost.Asio 是 Boost C++ 库中的一个跨平台网络与底层I/O库，主要用于处理 网络通信、异步I/O、定时器等。它是 C++ 中非常流行的网络编程库之一。

asio异步读写案例

Session类

Session类是处理客户端消息收发的会话类。简单起见，没有考虑粘包问题，仅采用简单的问答方式收发固定长度的数据。

_data用来接收收发的数据

_sock每个会话类单独通讯的socket

handle_write和hanle_read分别是发送和接收的回调函数

隐患

该demo示例为仿照asio官网编写的，其中存在隐患，就是当服务器即将发送数据前(调用async_write前)，此刻客户端中断，服务器此时调用async_write会触发发送回调函数，判断ec为非0进而执行delete this逻辑回收session。但要注意的是客户端关闭后，在tcp层面会触发读就绪事件，服务器会触发读事件回调函数。在读事件回调函数中判断错误码ec为非0，进而再次执行delete操作，从而造成二次析构，这是极度危险的。

利用C11模拟伪闭包实现连接的安全回收

我们使用伪闭包的方式来规避上面代码的隐患

构造伪闭包

  1. 利用智能指针被复制或使用引用计数加一的原理保证内存不被回收
  2. bind操作可以将值绑定在一个函数对象上生成新的函数对象，如果将智能指针作为参数绑定给函数对象，那么智能指针就以值的方式被新函数对象使用，那么智能指针的生命周期将和新生成的函数对象一致，从而达到延长生命的效果。我们按照上面的思路改写我们的回调函数。

Session类

class Session: public std::enable_shared_from_this<Session>
&#123;
public:
    Session(asio::io_context &ios,Server* server);
    asio::ip::tcp::socket& Socket();
    void Start();
    std::string& GetUuid();
private:
    void handle_write(std::shared_ptr<Session> self_shared,const system::error_code& error);
    void handle_read(std::shared_ptr<Session> self_shared,const system::error_code& error,std::size_t bytes_transfered);
    
    enum Size &#123; max_size = 1024 &#125;;
    char _data[max_size] = &#123; 0 &#125;;
    asio::ip::tcp::socket _sock;
    Server* _server;
    std::string _uuid;
&#125;;

在handle_write和handle_read中增加一个指向本对象的智能指针作为参数，在实际调用时，使用bind绑定函数对象，每传递一次，就会使智能指针引用计数加一，因此确保了每个回调函数执行时都至少有一个引用存在，从而确保了还有对象存在。


void Session::handle_write(std::shared_ptr<Session> self_shared,const system::error_code& error)
&#123;
    if (!error) &#123;
        memset(_data, 0, max_size);
        _sock.async_read_some(asio::buffer(_data, max_size), std::bind(&Session::handle_read, this,self_shared,
            std::placeholders::_1, std::placeholders::_2));
    &#125;
    else &#123;
        _server->ClearSession(_uuid);
    &#125;
&#125;
void Session::handle_read(std::shared_ptr<Session> self_shared,const system::error_code& error, std::size_t bytes_transfered)
&#123;
    if (!error) &#123;
        std::cout << "_data is:" << _data<<std::endl;
        //这里write保证发完，所以回调函数只需要一个错误代码作为参数
        asio::async_write(_sock,asio::buffer(_data, max_size), 
            std::bind(&Session::handle_write, this,self_shared,std::placeholders::_1));
    &#125;
    else &#123;
        _server->ClearSession(_uuid);
    &#125;
&#125;

shared_ptr<CSession>(this)生成的新智能指针和this之前绑定的智能指针并不共享引用计数，所以要通过shared_from_this()函数返回智能指针，该智能指针和其他管理这块内存的智能指针共享引用计数。

void Session::Start() &#123;
        _sock.async_read_some(asio::buffer(_data, max_size),
        std::bind(&Session::handle_read, this,shared_from_this(), std::placeholders::_1, std::placeholders::_2));
&#125;

shared_from_this()函数并不是session的成员函数，要使用这个函数需要继承std::enable_shared_from_this<Session>
